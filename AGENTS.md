## Review guidelines

### 1. 언어

- 모든 리뷰 코멘트는 한국어로 작성한다.
- 코드 식별자, 함수명, 클래스명은 원문 그대로 유지한다.

### 2. 리뷰 구조

리뷰는 아래 순서를 따른다:

1) 변경 사항 요약
2) 문제점 (심각도 표시)
3) 개선 제안
4) 선택적 개선 사항 (Optional)

### 3. 심각도 기준

- P0: 배포 불가 수준 (버그, 보안, 데이터 손상 가능성)
- P1: 수정 권장 (성능, 예외 처리 누락, 잠재적 오류)
- P2: 개선 제안 (가독성, 리팩토링, 네이밍)

### 4. 리뷰 스타일

- 불필요한 장황한 설명은 지양한다.
- 구체적인 수정 예시 코드를 포함한다.
- “좋아요” 같은 의미 없는 코멘트는 작성하지 않는다.
- 확실하지 않은 부분은 추측하지 말고 질문 형태로 작성한다.
